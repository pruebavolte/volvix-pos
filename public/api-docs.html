Volvix POS · API

v1 Base URL: https://<tu-dominio>

Referencia de los endpoints HTTP/JSON expuestos por el backend serverless de Volvix POS. Todos los endpoints autenticados requieren Authorization: Bearer <jwt>. El header X-Tenant-Id es opcional (se infiere del token si está ausente). Las respuestas son JSON salvo donde se indica explícitamente PDF/XML.

Autenticación

POST/api/auth/loginpúblico
Intercambia email/password por un JWT y datos de sesión.
Request & Response 
{
  "email": "owner@tienda.com",
  "password": "••••••••"
}
{
  "ok": true,
  "access_token": "eyJ...",
  "user": { "id": "u_1", "email": "...", "role": "owner", "tenant_id": "t_1" },
  "expires_at": 1714521600000
}
POST/api/auth/registerpúblico
Crea un tenant + usuario owner. Usa /api/email/otp para verificar correo.
POST/api/auth/refreshpúblico
Renueva el JWT a partir de un refresh_token vigente.
GET/api/auth/meauth
Devuelve el usuario y tenant asociado al token activo.
GET/api/healthpúblico
Latido del servicio. { ok: true, version: "..." }.

Productos

GET/api/productsauth
Lista productos del tenant. Soporta ?q=, ?low_stock=1, ?limit=, ?branch_id=. Alias en español: /api/productos.
Response 
[
  { "id":"p_1","name":"Coca 600ml","code":"7501055303045","price":25,"stock":124 }
]
POST/api/productsauth
Crea un producto. Detecta duplicados por code (sugiere force_create=true o merge).
Body 
{ "name":"Pan dulce","code":"PD-001","price":8.5,"cost":4,"stock":50,"category":"Panadería" }
PATCH/api/products/:idauth
Actualiza campos parciales. Mantiene cost-history al cambiar cost.
DELETE/api/products/:idauth
Soft-delete del producto.
POST/api/products/importauth
Importación transaccional CSV/JSON. Inserta todo o nada.
GET/api/products/by-barcodeauth
Búsqueda por código de barras: ?barcode=7501....
POST/api/products/:id/mergeauth
Fusiona dos SKUs en uno. ?into=<other_id>.

Ventas

GET/api/salesauth

POST/api/salesauth
Crea una venta. Soporta idempotency-key. Alias: /api/ventas.
Body 
{
  "items":[{"product_id":"p_1","qty":2,"price":25}],
  "payment_method":"efectivo",
  "total":50,
  "customer_id":"c_1"

GET/api/sales/:idauth
Detalle de una venta con items y pagos asociados.
POST/api/sales/:id/refundauth
Devolución total/parcial. Reabastece inventario.
POST/api/sales/emergency-syncauth
Sube ventas en cola desde el modo offline.

Status / Uptime

GET/api/status/incidentspúblico
Incidentes activos/recientes (estilo statuspage).
GET/api/status/uptimepúblico
Métricas de uptime por servicio (últimos 90 días).
POST/api/status/incidentsauth (admin)
Crea incidente. Body: { service, severity, message }.
PATCH/api/status/incidents/:idauth (admin)
Actualiza un incidente.
POST/api/status/incidents/:id/resolveauth (admin)
Marca incidente como resuelto.
POST/api/status/ping/:servicepúblico
Ping desde el monitor externo. Marca el servicio vivo.
Convenciones generales: Errores siguen el formato { ok:false, error:"...", code:"..." }. HTTP 401 = sesión expirada (cliente debe re-loguear). HTTP 503 = servicio sin configurar (incluye env_var faltante). Todos los POST mutables aceptan Idempotency-Key en header.
