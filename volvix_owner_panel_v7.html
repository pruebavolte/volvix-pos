Motor de sincronización
Cómo mantenemos los 3 clients alineados

1. Modelo de datos

Cada entidad (venta, producto, cliente) usa UUID + vector clock. Los conflictos se resuelven con last-write-wins por campo, salvo en inventario donde aplicamos sumas atómicas (CRDT counter).

2. Offline-first

Toda operación se escribe primero a la base local (IndexedDB / SQLite / Room) y luego se encola para sync. El usuario nunca espera a la red para cobrar un ticket.

3. Reconciliación

Cuando vuelve la conexión, cada cliente envía su outbox y recibe los cambios desde el último sync_token. El servidor es source of truth, pero nunca bloqueante.

4. UI idéntica

Las 3 apps consumen el mismo design system y los mismos contratos de datos. Si ves una venta en Windows, se ve igual en Web y en Android en segundos.
