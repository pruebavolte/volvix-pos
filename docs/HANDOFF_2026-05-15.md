## 🚨 Problemas conocidos / Bugs activos

### 1. UAC repetido al instalar

Cada upgrade install requiere aceptar UAC. NO hay forma de eliminarlo completamente (Windows lo exige por seguridad). El usuario lo aceptó múltiples veces hoy.

### 2. BT pairing requiere notificación toast del SO

En Windows 11, aunque mi código auto-empareja vía WinRT, el SO puede mostrar un toast notification que el usuario debe aceptar UNA vez. NO sale el wizard tradicional.

### 3. Diálogo "Seleccionar impresora" salía en versiones viejas

En screenshot del usuario v1.0.301 aparecía. v1.0.315 elimina esto con `silent: true` + combobox + persistencia de selección.

### 4. Volvix POS a veces no abre tras install

Tras `Start-Process` el .exe puede crashear si Electron caches están corruptos. Solución: matar todos los procesos `Volvix POS`, esperar 5s, abrir de nuevo desde icono escritorio.

### 5. Tab Config tarda en mostrar editor en vivo

El editor (`VolvixTicketEditor.renderInto`) se inicializa en el `showCfg('impresion')` hook. Si el usuario abre Config muy rápido, puede no estar listo. Setting timeout 50ms.

## 🧭 Filosofía del proyecto

**Usuario target: adulto mayor 60-75 años, NO técnico.**

Reglas:
1. **CERO clicks** después del install (1 UAC). Todo automático.
2. **Errores con lenguaje claro y visual**: modal rojo gigante "📄 Se acabó el papel" no "Error 0x80004005".

4. **Reintentar antes de mostrar error**: auto-repair USB, retry BT, fallback IP→USB.
5. **NUNCA descargar drivers externos**: usar `Generic / Text Only` (built-in Windows).
6. **NUNCA mostrar diálogo de Windows** para imprimir: `silent: true` + combobox guardado.
7. **Soporte remoto sin descargar nada**: WebRTC + nut-js desde dentro de la app.

## 📝 Cómo extender el sistema

### Agregar un campo al ticket
1. Edita `public/volvix-ticket-customizer.js`:
   - Agregá la key en `DEFAULT_CONFIG`
   - Agregá la lógica en `renderText(data, cfg)`
2. Edita `public/volvix-ticket-editor.js`:
   - Agregá un `buildToggle()` o `buildField()` en la sección correspondiente
3. El preview en vivo se actualiza automáticamente

### Agregar un método de impresión nuevo
1. Crea `electron/printer-XXX.js` con función `printTicketXXX(opts)`
2. Agrega `require()` en `electron/main.js`
3. Agrega IPC handler `volvix:XXX:print`
4. Expone en `electron/preload.js` como `window.volvixElectron.printXXX`
5. Agrega caso en `autoPrintTicket()` en `volvix-cobro-modal.js`
6. Agrega selector en `volvix-print-config.js`
